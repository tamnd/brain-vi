# 1. CPython là gì 

CPython là triển khai chính của ngôn ngữ lập trình Python. Đây là cách triển khai mà hầu hết mọi người sử dụng khi họ cài đặt Python từ python.org, trình quản lý gói, bản phân phối Linux, Homebrew, pyenv, hình ảnh Docker hoặc hầu hết các thời gian chạy trên đám mây. 

CPython có hai vai trò. 

Đầu tiên, nó triển khai ngôn ngữ Python. Nó xác định cách mã nguồn Python được phân tích, biên dịch, thực thi và kết nối với thư viện chuẩn. 

Thứ hai, nó đóng vai trò là điểm tham chiếu thực tế cho hệ sinh thái Python. Các cách triển khai khác có thể tuân theo đặc tả ngôn ngữ, nhưng CPython là cách triển khai mà hầu hết các gói, công cụ, trình gỡ lỗi, trình lược tả và phần mở rộng C đều được thử nghiệm. 

Cây nguồn CPython chứa trình thông dịch, trình biên dịch, hệ thống đối tượng, thời gian chạy, API C, thư viện chuẩn, bộ thử nghiệm, tài liệu và máy xây dựng. Hướng dẫn chính thức dành cho nhà phát triển mô tả CPython là cơ sở mã thay đổi với tài liệu nội bộ được duy trì trong cây nguồn. ([Hướng dẫn dành cho nhà phát triển Python] [1]) 

## 1.1 Ngôn ngữ Python so với việc triển khai CPython 

Python là ngôn ngữ. 

CPython là một trong những triển khai của ngôn ngữ đó. 

Sự khác biệt này quan trọng vì ngôn ngữ Python mô tả hành vi hiển thị: cú pháp, tên, biểu thức, câu lệnh, đối tượng, mô-đun, ngoại lệ, lặp lại, hành vi không đồng bộ, nhập, v.v. 

CPython mô tả một cách cụ thể để làm cho hành vi đó chạy trên máy thật. 

Một chương trình Python như thế này:```python
def add(a, b):
    return a + b

print(add(2, 3))
```là Python cấp độ ngôn ngữ. Ngôn ngữ cho chúng ta biết rằng`def`tạo một đối tượng hàm,`return`trả về một giá trị,`+`gọi ngữ nghĩa bổ sung, và`print`viết một biểu diễn văn bản. 

CPython thực hiện các hoạt động đó thông qua các cơ chế cấp thấp hơn:```text
source text
    ↓
tokens
    ↓
parser
    ↓
abstract syntax tree
    ↓
symbol table
    ↓
code object
    ↓
bytecode
    ↓
evaluation loop
    ↓
object operations
    ↓
C functions and machine code
```Lập trình viên thường chỉ nhìn thấy phần trên cùng của đường ống này. Các nhà phát triển CPython làm việc trên toàn bộ quy trình. 

## 1.2 CPython vừa là trình biên dịch vừa là trình thông dịch 

CPython thường được gọi là thông dịch viên. Điều đó đúng nhưng chưa đầy đủ. 

CPython trước tiên biên dịch mã nguồn Python thành mã byte. Sau đó nó diễn giải mã byte đó. 

Giai đoạn biên dịch không tạo ra mã máy gốc theo nghĩa trình biên dịch trước thời hạn thông thường. Nó tạo ra`code`các đối tượng chứa hướng dẫn mã byte, hằng số, tên, thông tin bố cục biến, bảng ngoại lệ, thông tin số dòng và siêu dữ liệu cần thiết cho thời gian chạy. 

Một tập tin như thế này:```python
x = 10
y = 20
print(x + y)
```được chuyển đổi thành mã byte có hình dạng gần giống như:```text
load constant 10
store name x
load constant 20
store name y
load name print
load name x
load name y
binary add
call
```Mã byte thực tế thay đổi trên các phiên bản Python, nhưng mô hình vẫn giữ nguyên: CPython hạ mã nguồn thành luồng lệnh nhỏ gọn và thực thi luồng đó trong máy ảo. 

Bạn có thể kiểm tra mã byte bằng`dis`:```python
import dis

def add(a, b):
    return a + b

dis.dis(add)
```Điều này cung cấp chế độ xem ở cấp độ Python về mô hình thực thi của CPython. 

## 1.3 Máy ảo CPython 

Máy ảo CPython là trình thông dịch mã byte dựa trên ngăn xếp. 

Hầu hết các hướng dẫn mã byte hoạt động trên ngăn xếp đánh giá. Hướng dẫn đẩy giá trị, giá trị pop, gọi hàm, tạo vùng chứa, nhánh, tăng ngoại lệ và trả về kết quả. 

Ví dụ: đánh giá:```python
a + b
```yêu cầu tải CPython`a`, trọng tải`b`và áp dụng phép cộng nhị phân. Về mặt khái niệm:```text
LOAD_FAST a      push local variable a
LOAD_FAST b      push local variable b
BINARY_OP +      pop two values, add them, push result
```Máy ngăn xếp này tồn tại bên trong khung hiện đang được thực thi. Một khung lưu trữ trạng thái thực thi của lệnh gọi hàm: biến cục bộ, ngăn xếp giá trị, con trỏ lệnh, đối tượng mã, toàn cục, nội dung và trạng thái ngoại lệ. 

Khi chạy, CPython liên tục thực hiện mẫu này:```text
fetch next bytecode instruction
decode instruction and operand
dispatch to implementation
operate on Python objects
update frame state
continue or return
```Vòng lặp trình thông dịch cốt lõi là một trong những phần quan trọng nhất của CPython. Nó kết nối mã byte với hành vi của đối tượng. 

## 1.4 CPython được viết chủ yếu bằng C và Python 

CPython được triển khai chủ yếu bằng C, với một lượng lớn mã Python trong thư viện chuẩn. 

Mã C triển khai thời gian chạy lõi:```text
object representation
reference counting
garbage collector
interpreter loop
compiler
parser support
built-in types
C API
threading primitives
import machinery support
memory allocators
```Mã Python triển khai phần lớn thư viện chuẩn:```text
collections
pathlib
asyncio
typing
unittest
importlib portions
dataclasses
logging
email
venv
many utility modules
```Một số mô-đun có cả triển khai Python và C. Phiên bản C thường được sử dụng để cải thiện hiệu năng, trong khi phiên bản Python có thể tồn tại để đảm bảo tính rõ ràng, tính di động, kiểm tra hoặc hành vi dự phòng. 

Thiết kế hỗn hợp này rất quan trọng. CPython không chỉ là một chương trình C chạy các tệp Python. Đó là thời gian chạy trong đó các thành phần cấp C và cấp Python hợp tác. 

## 1.5 CPython Xác định mô hình đối tượng trong bộ nhớ 

Mọi giá trị trong Python đều là một đối tượng. 

Trong CPython, mọi đối tượng đều bắt đầu bằng tiêu đề đối tượng cấp C. Ở mức tối thiểu, tiêu đề đó lưu trữ số lượng tham chiếu và con trỏ tới loại đối tượng. 

Về mặt khái niệm:```c
typedef struct {
    Py_ssize_t ob_refcnt;
    PyTypeObject *ob_type;
} PyObject;
```Các đối tượng có kích thước thay đổi sử dụng tiêu đề lớn hơn:```c
typedef struct {
    PyObject ob_base;
    Py_ssize_t ob_size;
} PyVarObject;
```Điều này có nghĩa là số nguyên, chuỗi, danh sách, bộ, hàm, lớp, mô-đun, ngoại lệ, khung và trình tạo Python đều có một con trỏ kiểu. Đối tượng loại mô tả những hoạt động mà đối tượng hỗ trợ. 

Ví dụ:```python
x = 42
```tạo hoặc tham chiếu đến một đối tượng số nguyên Python. CPython không lưu trữ`x`dưới dạng số nguyên CPU thô trong mã Python thông thường. Nó lưu trữ một tham chiếu đến một`PyLongObject`. 

Sự khác biệt đó giải thích nhiều hành vi của Python:```python
x = 42
y = x
```Đây,`x`Và`y`đề cập đến cùng một đối tượng. Bài tập liên kết tên với các đối tượng. Nó không sao chép đối tượng theo mặc định. 

## 1.6 Kiểu cũng là đối tượng 

Trong CPython, kiểu cũng là đối tượng. 

Biểu thức:```python
type(42)
```trả về đối tượng đại diện`int`. 

Biểu thức:```python
type(int)
```trả lại`type`. 

Đây không chỉ là một thủ thuật ngôn ngữ. CPython đại diện cho các loại bằng cách sử dụng`PyTypeObject`, một cấu trúc C lớn lưu trữ siêu dữ liệu kiểu và con trỏ hàm được gọi là slot. 

Một đối tượng kiểu trả lời các câu hỏi như:```text
How large is each instance?
How should instances be deallocated?
How is attribute lookup performed?
How is the object called?
How does addition work?
How does indexing work?
How does iteration work?
How is the object represented as text?
```Đối với một danh sách, đối tượng kiểu trỏ tới các triển khai dành riêng cho danh sách. 

Đối với một lệnh, nó trỏ đến việc triển khai dành riêng cho lệnh. 

Đối với lớp do người dùng định nghĩa, CPython xây dựng một đối tượng kiểu một cách linh hoạt. 

Đây là lý do tại sao mô hình dữ liệu của Python ánh xạ rõ ràng vào phần bên trong CPython. Các phương pháp như`__len__`,`__getitem__`,`__call__`,`__iter__`, Và`__add__`tương ứng với các khe hoạt động nội bộ. 

## 1.7 Tên là tài liệu tham khảo 

Biến Python là tên được liên kết với các đối tượng. 

CPython thực hiện điều này thông qua tài liệu tham khảo. 

Trong Python:```python
a = [1, 2, 3]
b = a
b.append(4)
print(a)
```Đầu ra là:```text
[1, 2, 3, 4]
```Cả hai`a`Và`b`tham chiếu đến cùng một đối tượng danh sách. 

Ở cấp độ CPython, bài tập`b = a`tăng số lượng tham chiếu của đối tượng danh sách. Nó không trùng lặp danh sách. 

Mô hình này ảnh hưởng đến hiệu suất và độ chính xác. Truyền danh sách cho hàm sẽ truyền tham chiếu đến đối tượng:```python
def push(xs):
    xs.append(4)

items = [1, 2, 3]
push(items)
```Hàm nhận được một tham chiếu khác tới cùng danh sách. 

## 1.8 Đếm tham chiếu là cơ chế quản lý bộ nhớ chính 

CPython chủ yếu sử dụng tính năng tham chiếu để lấy lại các đối tượng. 

Mỗi đối tượng ghi lại số lượng tham chiếu mạnh trỏ đến nó. Khi số đếm về 0, CPython sẽ giải phóng đối tượng ngay lập tức. Tài liệu Python mô tả trình thu gom rác như một phần bổ sung cho việc đếm tham chiếu, chủ yếu để phát hiện các chu trình tham chiếu không thể truy cập được. ([Tài liệu Python][2]) 

Về mặt khái niệm:```text
create object         refcount = 1
bind another name     refcount += 1
delete a name         refcount -= 1
refcount reaches 0    destroy object
```Ví dụ:```python
x = []
y = x
del x
del y
```Danh sách tồn tại sau`del x`bởi vì`y`vẫn đề cập đến nó. Sau đó`del y`, số lượng tham chiếu có thể đạt tới 0, vì vậy CPython có thể giải phóng danh sách. 

Việc đếm tham chiếu mang lại khả năng hoàn thiện đối tượng có thể dự đoán được của CPython trong nhiều trường hợp. Ví dụ: một đối tượng tệp có thể đóng ngay khi tham chiếu cuối cùng của nó biến mất. Nhưng việc tính tham chiếu một mình không thể xử lý các chu kỳ:```python
a = []
b = []
a.append(b)
b.append(a)
```Đây,`a`Và`b`tham khảo lẫn nhau. Ngay cả khi không có tên bên ngoài nào trỏ tới chúng, số lượng tham chiếu của chúng có thể vẫn khác 0. Trình thu gom rác theo chu kỳ của CPython tồn tại để tìm và dọn sạch các chu kỳ đó. Các loại vùng chứa có thể tham gia vào chu trình cần có sự hỗ trợ rõ ràng của GC. ([Tài liệu Python][3]) 

## 1.9 CPython có Trình thu gom rác theo chu kỳ 

Trình thu gom rác tuần hoàn xử lý các nhóm đối tượng tham chiếu lẫn nhau nhưng không thể truy cập được từ gốc chương trình trực tiếp. 

Nó chủ yếu theo dõi các đối tượng vùng chứa: danh sách, ký tự, bộ, bộ dữ liệu chứa tham chiếu, hàm, lớp, khung và các đối tượng khác có thể trỏ đến các đối tượng khác. 

Các đối tượng nguyên tử như nhiều số nguyên và chuỗi thường không cần theo dõi GC theo chu kỳ vì chúng không chứa các tham chiếu đến các đối tượng Python khác. 

Bộ sưu tập hoạt động cùng với việc đếm tham chiếu:```text
reference counting handles most objects immediately
cyclic GC handles unreachable reference cycles
```Thiết kế lai này định hình hành vi CPython. Trí nhớ thường được giải phóng nhanh chóng nhưng một số cấu trúc tuần hoàn được thu thập sau đó. 

các`gc`mô-đun tiết lộ một phần của máy móc này:```python
import gc

gc.collect()
print(gc.get_count())
print(gc.isenabled())
```Mô-đun này hữu ích khi nghiên cứu vòng đời của đối tượng, rò rỉ, hoàn thiện và hành vi chu trình. 

## 1.10 CPython sử dụng Khóa phiên dịch toàn cầu 

CPython truyền thống sử dụng Khóa phiên dịch toàn cầu, thường được gọi là GIL. 

GIL chỉ cho phép một luồng tại một thời điểm thực thi mã byte Python trong một trình thông dịch. Điều này giúp đơn giản hóa việc quản lý bộ nhớ và bảo vệ trạng thái trình thông dịch nội bộ, đặc biệt là các cập nhật về số lượng tham chiếu. 

Điều này không có nghĩa là CPython không thể sử dụng các luồng. Điều đó có nghĩa là mã byte Python gắn với CPU thường không chạy song song trên nhiều luồng hệ điều hành trong cùng một trình thông dịch. 

Chủ đề vẫn có thể hữu ích cho:```text
I/O-bound programs
network clients
file operations
concurrent waiting
C extensions that release the GIL
```Đối với tính song song giới hạn CPU, các chương trình Python thường sử dụng:```text
multiprocessing
native extensions
vectorized libraries
subinterpreters in newer designs
free-threaded builds in modern CPython work
```GIL là một chi tiết triển khai kèm theo các hậu quả ở cấp độ ngôn ngữ. Mã xuất hiện đồng thời ở cấp Python có thể tuần tự hóa trong quá trình thực thi mã byte. 

## 1.11 CPython có API C công khai 

CPython hiển thị API C cho các mô-đun mở rộng và nhúng. 

Đây là một trong những tính năng hệ sinh thái quan trọng nhất của CPython. Các thư viện như NumPy, pandas, lxml, mật mã, Gối và nhiều trình điều khiển cơ sở dữ liệu phụ thuộc vào cơ chế mở rộng riêng. 

Một chức năng mở rộng tối thiểu có thể nhận được`PyObject *`giá trị, phân tích các đối số, gọi mã C và trả về một giá trị mới`PyObject *`. 

Tại ranh giới này, quyền sở hữu tài liệu tham khảo là rất quan trọng. CPython phân biệt tài liệu tham khảo mượn, tài liệu tham khảo mới và tài liệu tham khảo bị đánh cắp. Việc xử lý quyền sở hữu không chính xác có thể gây rò rỉ bộ nhớ, lỗi use-after-free, sự cố hoặc trạng thái trình thông dịch bị hỏng. 

Ví dụ về khái niệm:```text
Py_INCREF     take another strong reference
Py_DECREF     release a strong reference
Py_XDECREF    release if pointer may be NULL
PyObject      base object pointer
PyTypeObject  type object structure
PyModuleDef   extension module definition
```API C làm cho CPython có khả năng mở rộng bất thường, nhưng nó cũng liên kết phần lớn hệ sinh thái với hành vi thời gian chạy và biểu diễn đối tượng của CPython. 

## 1.12 CPython không giống với ngữ nghĩa của Python 

Một số hành vi CPython là chi tiết triển khai. 

Ví dụ: việc phá hủy đối tượng ngay lập tức thông qua việc đếm tham chiếu là hành vi CPython. Mã Python thường nên tránh phụ thuộc vào thời gian hủy chính xác trừ khi hành vi đó được đảm bảo rõ ràng bởi ngôn ngữ hoặc bởi trình quản lý ngữ cảnh. 

Đây là di động:```python
with open("data.txt") as f:
    data = f.read()
```Đây là thời gian cụ thể của CPython:```python
f = open("data.txt")
data = f.read()
f = None
```Trong Python, việc gán`None`có thể đóng tệp ngay lập tức nếu nó xóa tham chiếu cuối cùng. Trong một triển khai Python khác, việc dọn dẹp có thể diễn ra sau. 

Trình đọc nội bộ CPython tốt phải liên tục tách ba lớp: 

| Lớp | Ví dụ | Tính ổn định | 
| -------------- | ---------------------------------------- | ---------------------------------- | 
| Ngữ nghĩa ngôn ngữ |`for`vòng lặp gọi giao thức lặp | Rất ổn định | 
| Hành vi CPython | Đối tượng sử dụng số tham chiếu | Ổn định nhưng cụ thể khi triển khai | 
| Bố cục nội bộ CPython | Các trường cấu trúc chính xác và mã bytecode | Có thể thay đổi giữa các phiên bản | 

## 1.13 CPython và các triển khai Python khác 

CPython không phải là triển khai Python duy nhất. 

Các lựa chọn thay thế phổ biến bao gồm: 

| Thực hiện | Cơ sở thời gian chạy | Sự khác biệt chính | 
| -------------- | ------------ | ---------------------------------------------------------------- | 
| CPython | C | Triển khai tham khảo và khả năng tương thích tiện ích mở rộng C mạnh nhất | 
| PyPy | RPython | Triển khai theo định hướng JIT | 
| Jython | JVM | Python trên nền tảng Java | 
| SắtPython | .NET | Python trên thời gian chạy .NET | 
| MicroPython | C | Tập hợp con Python cho bộ vi điều khiển và các thiết bị bị ràng buộc | 

Hầu hết người dùng Python đều chạy CPython. Hầu hết các gói Python đều được thử nghiệm đầu tiên trên CPython. Hầu hết các thư viện tiện ích mở rộng gốc đều nhắm mục tiêu API C của CPython. 

Điều này làm cho CPython trở thành trung tâm ngay cả khi nghiên cứu Python như một ngôn ngữ. 

## 1.14 CPython thực thi những gì 

CPython không thực thi văn bản nguồn trực tiếp. 

Nó thực thi các đối tượng mã. 

Một đối tượng mã chứa mã byte và siêu dữ liệu được biên dịch:```text
bytecode instruction stream
constants
names
local variable names
free variables
cell variables
argument counts
stack size
filename
function name
line number table
exception table
flags
```Bạn có thể kiểm tra đối tượng mã của hàm:```python
def f(x):
    y = x + 1
    return y

code = f.__code__

print(code.co_name)
print(code.co_varnames)
print(code.co_consts)
print(code.co_names)
```Một đối tượng hàm bao bọc một đối tượng mã cùng với bối cảnh thời gian chạy:```text
code object
globals dictionary
defaults
keyword defaults
closure cells
annotations
qualname
module
```Sự khác biệt này quan trọng. Cùng một đối tượng mã có thể được sử dụng lại với các đối tượng hàm, toàn cục, mặc định hoặc đóng khác nhau. 

## 1.15 Đơn vị thực thi chính của CPython 

Một tiến trình CPython đang chạy chứa một số đơn vị thực thi quan trọng. 

| Đơn vị | Ý nghĩa | 
| ------------ | -------------------------------------------------------------------------------- | 
| Quy trình | Quá trình hệ điều hành chạy CPython | 
| Thời gian chạy | Trạng thái thời gian chạy CPython toàn cầu | 
| Thông dịch viên | Trạng thái trình thông dịch với các mô-đun, nội dung, trạng thái nhập và bối cảnh thực thi | 
| Trạng thái chủ đề | Trạng thái thực thi trên mỗi luồng | 
| Khung | Thực thi một hàm/mô-đun/lớp đang hoạt động | 
| Đối tượng mã | Mã byte và siêu dữ liệu được biên dịch | 
| Đối tượng | Giá trị thời gian chạy được thao tác bằng mã byte | 

Một lệnh gọi hàm đơn giản sẽ tạo hoặc sử dụng lại một khung. Khung đó thực thi một đối tượng mã. Khung thuộc về trạng thái luồng. Trạng thái luồng thuộc về một trình thông dịch. Trình thông dịch thuộc về thời gian chạy. 

Về mặt khái niệm:```text
CPython process
    runtime
        interpreter
            thread state
                frame
                    code object
                    value stack
                    locals
```Hệ thống phân cấp này xuất hiện xuyên suốt mã CPython C. 

## 1.16 Thư viện chuẩn của CPython là một phần của quá trình triển khai 

Thư viện chuẩn không tách rời khỏi CPython trong thực tế. 

Các mô-đun như`sys`,`types`,`gc`,`inspect`,`dis`,`importlib`, Và`ctypes`hiển thị hoặc tương tác với nội bộ thời gian chạy. 

Ví dụ:```python
import sys
print(sys.getrefcount(object()))

import dis
dis.dis(lambda x: x + 1)

import gc
print(gc.get_objects())

import inspect
print(inspect.currentframe())
```Một số mô-đun là Python thuần túy. Một số được triển khai trong C. Một số sử dụng cả hai. 

Ranh giới giữa trình thông dịch và thư viện chuẩn là có chủ ý thực tế. CPython đặt các hoạt động cấp thấp trong C khi cần thiết và điều phối cấp độ cao hơn trong Python khi có thể. 

## 1.17 CPython có bộ thử nghiệm lớn 

CPython bao gồm một bộ thử nghiệm hồi quy lớn. 

Bộ thử nghiệm phục vụ một số mục đích:```text
protect language behavior
protect standard library behavior
catch platform-specific bugs
test C API behavior
test compiler and parser behavior
test memory management behavior
test edge cases from historical bugs
```Đối với công việc nội bộ, bộ thử nghiệm không phải là tùy chọn. Các thay đổi của CPython có thể âm thầm ảnh hưởng đến việc phân tích cú pháp, mã byte, vòng đời của đối tượng, quá trình nhập, ngoại lệ, phân luồng, hành vi không đồng bộ hoặc khả năng tương thích của tiện ích mở rộng. 

Một vòng phát triển bình thường trông như sau:```text
edit source
build CPython
run targeted tests
run broader tests
check reference leaks if relevant
inspect performance if relevant
```## 1.18 Tại sao nội bộ CPython lại quan trọng 

Nội bộ CPython đóng vai trò quan trọng đối với một số loại công việc.

Đối với những người đóng góp CPython, nội bộ là tài liệu làm việc: bố cục đối tượng, vòng đánh giá, trình biên dịch, bộ cấp phát bộ nhớ, trừu tượng hóa nền tảng, kiểm tra và tài liệu. 

## 1.19 Một mô hình tinh thần cho phần còn lại của cuốn sách 

Sử dụng mô hình này trong khi đọc mã nguồn CPython:```text
Python code is compiled into code objects.
Code objects contain bytecode.
Frames execute code objects.
The evaluation loop executes bytecode instructions.
Bytecode instructions manipulate PyObject pointers.
Objects point to type objects.
Type objects define behavior through slots.
Memory is managed mainly through reference counts.
The cyclic GC cleans up unreachable cycles.
The C API exposes much of this machinery to extensions.
```Mô hình đó đủ để định hướng hầu hết mọi hệ thống con CPython. 

## 1.20 Tóm tắt chương 

CPython là triển khai chính của Python. Nó biên dịch mã nguồn Python thành mã byte, thực thi mã byte đó trong máy ảo dựa trên ngăn xếp, biểu thị tất cả các giá trị thời gian chạy dưới dạng đối tượng, quản lý bộ nhớ chủ yếu thông qua tính tham chiếu, bổ sung tính năng thu thập rác theo chu kỳ và hiển thị API C mạnh mẽ. 

Phần còn lại của cuốn sách coi CPython như một hệ thống cụ thể: trình biên dịch, máy ảo, thời gian chạy đối tượng, trình quản lý bộ nhớ, nền tảng mở rộng và triển khai thư viện tiêu chuẩn.
